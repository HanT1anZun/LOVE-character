4. 输出内容要适合高校课程项目、调研展示、轻量研究型产品落地。

【理论框架要求】
请不要直接照搬任何现成量表的原题；请使用“科研构念启发 + 原创改写题项”的方式，形成可上线的自定义问卷。

问卷建议围绕以下成熟构念建立：

- 关系沟通偏好与冲突处理风格

【结果类型设计要求】
请把最后的结果设计成“恋爱人格画像”，不是诊断，不要出现“异常”“病理”“心理疾病”等表述。

- 理性规划型

- 高敏依恋型
- 自由探索型
- 回避独立型
- 矛盾拉扯型

要求：
1. 每个结果类型都要有：
   - 类型名
   - 一句话定义
   - 关系中的优势
   - 关系中的盲点
   - 适合的相处方式
   - 适合的伴侣偏好
   - 代表性行为特征
2. 每个用户的结果必须同时显示：
   - 主类型
   - 次类型
   - 各维度百分比
   - 置信度或一致性分数
   - 简短解释
3. 支持“结果接近时显示混合型”逻辑，不要强行硬分。

1. 题量建议 24–40 题，优先 32 题。
2. 采用 5 级李克特量表：
   - 非常不同意

   - 一般
   - 同意
   - 非常同意
3. 题目要均衡分布到各维度。
4. 每个题目都要有元数据：

   - subscale

   - source_construct
   - note
5. 题目尽量避免双重否定、模糊表达、绝对化措辞。
6. 题目语言要自然、适合中国高校年轻用户阅读。
7. 必须保留“反向题”，但数量不要太多，避免用户阅读负担。
8. 题库必须输出为 Claude Code 可直接读取的 JSON 文件。

【科学性与可用性要求】

6. 如果同一维度分差很小，则显示“混合倾向”，不要强行给唯一人格。

【评分规则要求】
请实现以下通用算法：
1. 每个题目答案采用 1–5 分。
2. reverse = true 时进行反向计分。

4. 用维度组合生成主类型和次类型。
5. 当两个结果接近时输出混合型。
6. 生成一致性分数：
   - 过于极端的“全选同分”
   - 反向题前后冲突
   - 同一维度内差异过大
7. 所有评分过程必须写成可读函数，便于审查和替换。
